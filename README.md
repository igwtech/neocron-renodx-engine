# neocron-renodx-engine

ReShade Add-on Edition `.addon32` that injects custom HLSL pixel shaders
into Neocron 2's rendering pipeline. **Tier 4** of the graphics remaster
plan — the engine-level work that goes beyond ReShade post-processing.

## What it does (v0.1.0)

- Hooks D3D9 `init_pipeline` / `bind_pipeline` to fingerprint every
  vertex and pixel shader the game creates by CRC32.
- Dumps every unique pixel shader bytecode to `Z:/tmp/neocron_ps_<crc>.dxbc`
  so we can disassemble offline and identify each one.
- At addon load, compiles a custom HLSL replacement for `world.ps` via
  `D3DCompile` from `d3dcompiler_47.dll` (loaded dynamically).
- Hooks `create_pipeline` to substitute the original `world.ps` bytecode
  (CRC `0x2ccf5eb7`) with our compiled replacement when the game creates
  it. Returning `true` from `create_pipeline` tells ReShade we modified
  the subobjects in place.
- Logs replacement count + per-frame world.ps draw count for diagnostics.

## Investigation findings (kept as in-source comments)

### Pixel shaders identified

From bytecode CTAB inspection + (VS, PS, Z) histogram correlation
during gameplay:

| CRC | Size | What it is | Constants / samplers |
|---|---|---|---|
| `0x2ccf5eb7` | 496 B | **`world.ps`** — lit world geometry | `samplerAlbedo` (s0), `samplerLightmap` (s1), `colorCorrection` (c0) |
| `0x96f566cb` | 416 B | generic textured pass — used by skinned mesh, alpha-blended world overlays, AND HUD elements | (smaller, simpler — just modulated texture sampling) |
| `0xeb1b9a91` | 448 B | always-on overlay — pairs only with the 1-quad-per-frame VS `0x92e9b61d` | (cursor / damage flash) |

### Vertex shader outputs (world.vs CRC `0x5925dce1`, decoded from bytecode)

```
oPos = local_to_screen_matrix · vertexPos       (c8..c11)
oFog = saturate((fog_end - dist) / (fog_end - fog_start))   (c4 = fog_start_end)
oD0  = vertex diffuse colour
oT0  = TEXCOORD0 = albedo UV
oT1  = TEXCOORD1 = lightmap UV
```

This means the matching PS receives `TEXCOORD0` for the albedo sampler
and `TEXCOORD1` for the lightmap sampler. Confirmed empirically by
swapping the replacement shader to "albedo + red tint only" — full
texture detail (brick patterns, panel seams, tile floors) showed
red-tinted on every world surface.

## Build

Prerequisites:

```bash
sudo pacman -S mingw-w64-gcc
```

The Makefile reuses ReShade SDK headers fetched by
`_renodx_hello_stack/` in the parent project.

```bash
cd src && make    # → bin/neocron-renodx-engine.addon32
```

## Deploy

Manual:

```bash
cp bin/neocron-renodx-engine.addon32 ~/Neocron2/
```

Requires `neocron-renodx` (or equivalent) installed first so ReShade
Add-on Edition is loaded as `~/Neocron2/d3d9.dll`. Without ReShade
Add-on, the `.addon32` does nothing.

## Compatibility

Validated end-to-end on the same stack as the rest of the remaster:

- Manjaro Linux + Wayland
- Wine 10.0 (staging) via GE-Proton 10-34
- DXVK 2.7.1 backend
- ReShade 6.7.3 Add-on Edition + Microsoft `d3dcompiler_47.dll`
- Neocron 2 Evolution client (i386)
